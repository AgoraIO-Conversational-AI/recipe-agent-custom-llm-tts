# Recipe Contract

This base recipe defines the reusable surface for a Python-backed Agora Conversational AI **custom-llm-tts** quickstart: a cascading STT → Custom LLM (audio output) pipeline behind a Next.js web client. The custom LLM returns PCM audio directly — no TTS step — using an OpenAI-compatible endpoint that developers replace with their own model.

## Recipe Role

- Role: `base` recipe (self-contained, clone-and-run; no `Extends` pin).
- Target audience: developers building a voice agent where the response audio is generated by their own model or pipeline (TTS, text + pre-recorded audio, neural vocoder, etc.).
- Reuse model: clone, expose the backend publicly, set env vars, run with the mock, then replace `generate_tone()` in `server/src/llm.py` with your real audio source.

## Recipe Scope

- Python FastAPI token generation and managed agent lifecycle.
- A cascading `DeepgramSTT` → `CustomLLM(output_modalities=["audio"])` vendor chain with an inert `MiniMaxTTS` satisfying the builder.
- An OpenAI-compatible audio endpoint (`llm.py`) mounted at `/audio/chat/completions` — returns PCM16/16kHz SSE.
- A zero-key sine-wave mock for immediate local testing (no API keys required for the audio pipeline).
- Next.js browser UI with RTC audio, RTM transcript/metrics, connection status.
- Rewrite-only `/api/*` browser facade hiding backend placement.
- Contract, proxy, FastAPI, and LLM smoke verification that need no live Agora calls.

## Baseline Implementation Guidance

Use this repo's source and progressive disclosure docs as the starting point. Do not recreate the Agora ConvoAI integration from memory — vendor schemas, SDK builder fields, token behavior, and RTM details drift. Copy verified patterns from this repo.

## Invariants

- Browser code calls only `/api/get_config`, `/api/startAgent`, and `/api/stopAgent` for the default flow.
- Next.js owns `/api/*` through rewrites only; no `web/app/api/**/route.ts` for agent/token logic.
- FastAPI owns token generation, `AGORA_APP_CERTIFICATE`, and agent lifecycle.
- `CustomLLM` is configured with `output_modalities=["audio"]`; Agora cloud skips TTS and plays the PCM audio directly.
- `.with_tts(MiniMaxTTS(...))` stays in `agent.py` to satisfy the agora-agents builder.
- `server/src/llm.py` has no `agora_agent` import (AST-checked by `test_llm_mount.py`).
- The audio endpoint's first SSE chunk must include `delta.audio.transcript`.
- `CUSTOM_LLM_URL` is required and must be publicly reachable (Agora cloud calls it).
- The backend issues one RTC+RTM-capable token for a concrete non-zero UID.

## Stable Contracts

| Contract | Stable shape |
| -------- | ------------ |
| Required backend env | `AGORA_APP_ID`, `AGORA_APP_CERTIFICATE`, `CUSTOM_LLM_URL`, `CUSTOM_LLM_API_KEY` |
| Optional backend env | `CUSTOM_LLM_MODEL`, `AGENT_GREETING`, `PORT` (env only) |
| Required web deploy env | `AGENT_BACKEND_URL` |
| `GET /api/get_config` | Query `channel?`, `uid?`; returns `data.app_id`, `data.token`, `data.uid`, `data.channel_name`, `data.agent_uid`. |
| `POST /api/startAgent` | Body `{ channelName, rtcUid, userUid, parameters? }`; returns `data.agent_id`, `data.channel_name`, `data.status`. |
| `POST /api/stopAgent` | Body `{ agentId }`; returns `{ code: 0, msg: "success" }`. |
| `POST /audio/chat/completions` | OpenAI-compatible; `stream: true`; returns SSE with transcript chunk + base64 PCM16/16kHz/mono chunks + `data: [DONE]`. |
| Success envelope | `{ "code": 0, "msg": "success", "data": ... }` where the route has data. |
| Verification entry points | `bun run verify:web`, `bun run verify:backend`, `bun run verify:backend:pytest`, `bun run verify:web:proxy`, `bun run verify:local:fastapi`, `bun run verify:local:llm`, `bun run verify:local`. |

## Internal / Subject to Change

- Visual layout, component composition, Tailwind classes, and assets under `web/src/components/`.
- Mock audio implementation in `generate_tone()` — it is explicitly designed to be replaced.
- Exact STT model (`nova-3`), VAD timing values, voice (`English_captivating_female1`), and greeting text, as long as they stay documented extension points.
- In-memory `Agent._sessions` details; the stable behavior is start by channel/user and stop by returned `agent_id`.
- Verification internals under `web/scripts/`; the stable surface is the root script names and what they assert.
- `agora-agents` SDK minor-version behavior; this recipe lower-bounds `>=2.3.0` but does not freeze every field.
